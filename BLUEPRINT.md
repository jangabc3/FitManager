# 헬스장 회원관리 시스템

## 1. 프로젝트 이름

**헬스장 회원관리 시스템**

헬스장, PT샵, 필라테스샵 같은 운동 시설에서 회원 정보, 회원권, 출석, 결제 상태, 만료일을 쉽게 관리할 수 있는 개인 웹 프로젝트입니다.

이 프로젝트는 처음 웹 앱을 만드는 사람도 이해하기 쉽도록, 복잡한 기능보다 실제 매장에서 자주 쓰는 기본 기능에 집중합니다.

## 2. 프로젝트 목표

이 프로젝트의 목표는 헬스장 직원이나 운영자가 회원을 체계적으로 관리할 수 있는 간단한 웹 서비스를 만드는 것입니다.

기존에는 엑셀, 종이 장부, 메신저 메모 등으로 회원을 관리하는 경우가 많습니다. 이런 방식은 회원권 만료일을 놓치거나, 미납 회원을 확인하기 어렵고, 출석 기록도 흩어지기 쉽습니다.

이 시스템은 다음 문제를 해결하는 것을 목표로 합니다.

- 회원 정보를 한 곳에서 관리합니다.
- 회원권 시작일과 만료일을 쉽게 확인합니다.
- 오늘 출석한 회원을 빠르게 체크합니다.
- 결제 완료 회원과 미납 회원을 구분합니다.
- 만료 예정 회원과 이미 만료된 회원을 한눈에 확인합니다.

디자인은 딱딱한 관리자 화면보다는 **부드럽고 밝은 흰색 배경**을 기본으로 합니다. 카드형 UI, 보기 쉬운 버튼, 넓은 여백을 사용해 초보자도 편하게 사용할 수 있는 화면을 목표로 합니다.

## 3. 핵심 기능

### 회원 등록

새로운 회원을 등록하는 기능입니다.

입력 정보:

- 이름
- 연락처
- 회원권 종류
- 시작일
- 만료일
- 결제금액
- 결제상태

예를 들어 `김민준` 회원이 `헬스 3개월` 회원권을 구매했다면, 이름과 연락처, 시작일, 만료일, 결제금액을 입력해 저장합니다.

### 회원권 관리

회원이 어떤 회원권을 이용 중인지 관리합니다.

관리 정보:

- 회원권 종류
- 시작일
- 만료일
- 만료일까지 남은 날짜
- 회원권 상태

회원권 상태 예시:

- 이용중
- 만료예정
- 만료

### 출석 체크

회원이 헬스장에 방문했을 때 출석을 기록하는 기능입니다.

관리 정보:

- 출석 날짜
- 최근 출석일
- 총 출석 횟수

직원은 회원 목록에서 `출석 체크` 버튼을 눌러 오늘 출석을 저장할 수 있습니다.

### 결제 상태 관리

회원의 결제 상태를 관리하는 기능입니다.

결제 상태:

- 완료
- 미납

미납 회원은 목록에서 눈에 잘 띄도록 표시합니다. 운영자는 미납 회원을 빠르게 확인하고 결제 완료 상태로 변경할 수 있습니다.

### 회원권 만료일 관리

회원권 만료일을 기준으로 회원 상태를 자동으로 구분합니다.

상태 기준:

- 만료일까지 8일 이상 남음: 이용중
- 만료일까지 7일 이하 남음: 만료예정
- 만료일이 지남: 만료

이 기능을 통해 재등록 안내가 필요한 회원을 쉽게 찾을 수 있습니다.

## 사용 흐름

사용자는 보통 다음 순서로 서비스를 이용합니다.

1. 대시보드에서 전체 회원 수, 만료 예정 회원 수, 미납 회원 수, 오늘 출석 수를 확인합니다.
2. 새 회원이 오면 회원 등록 폼에 기본 정보와 회원권 정보를 입력합니다.
3. 회원이 방문하면 회원 목록 또는 출석 관리 영역에서 출석 체크를 합니다.
4. 결제가 필요한 회원은 결제 상태를 `미납`으로 표시합니다.
5. 결제가 완료되면 결제 상태를 `완료`로 변경합니다.
6. 만료 예정 또는 만료 회원을 확인하고 재등록 상담을 진행합니다.
7. 회원 정보가 바뀌면 수정 버튼을 눌러 정보를 변경합니다.
8. 더 이상 관리하지 않는 회원은 삭제할 수 있습니다.

## 4. 화면 구성

### 상단 헤더

서비스 이름과 전체 검색창이 있는 영역입니다.

주요 역할:

- 현재 서비스가 무엇인지 보여줍니다.
- 이름 또는 연락처로 회원을 빠르게 검색합니다.

### 좌측 사이드바 메뉴

주요 화면으로 이동할 수 있는 메뉴입니다.

메뉴 예시:

- 대시보드
- 회원 등록
- 회원 목록
- 출석 관리
- 결제 관리

### 대시보드 요약 카드

운영자가 가장 먼저 확인해야 할 정보를 카드 형태로 보여줍니다.

카드 항목:

- 전체 회원 수
- 만료 예정 회원 수
- 미납 회원 수
- 오늘 출석 수

### 회원 등록 폼

새 회원을 등록하거나 기존 회원 정보를 수정하는 화면입니다.

필드:

- 이름
- 연락처
- 회원권 종류
- 시작일
- 만료일
- 결제금액
- 결제상태

### 회원 목록 테이블

등록된 회원을 표 형태로 보여주는 화면입니다.

표시 정보:

- 이름
- 연락처
- 회원권
- 만료일
- 남은 기간
- 결제상태
- 최근 출석일
- 총 출석 횟수

가능한 작업:

- 검색
- 수정
- 삭제
- 결제 상태 변경

### 출석 관리 영역

회원별 출석을 체크하는 화면입니다.

가능한 작업:

- 오늘 출석 체크
- 오늘 출석 취소
- 최근 출석일 확인
- 총 출석 횟수 확인

## 5. 데이터 구조

처음에는 하나의 회원 데이터 안에 필요한 정보를 모두 담아도 충분합니다. 나중에 프로젝트가 커지면 회원, 회원권, 출석, 결제를 각각 분리할 수 있습니다.

### Member

회원 한 명을 나타내는 데이터입니다.

```js
{
  id: "고유 ID",
  name: "김민준",
  phone: "010-1234-5678",
  membershipType: "헬스 3개월",
  startDate: "2026-07-01",
  endDate: "2026-09-30",
  paymentAmount: 270000,
  paymentStatus: "완료",
  attendanceDates: ["2026-07-01", "2026-07-03"]
}
```

### 필드 설명

`id`: 회원을 구분하기 위한 고유 값입니다.

`name`: 회원 이름입니다.

`phone`: 회원 연락처입니다.

`membershipType`: 회원권 종류입니다.

`startDate`: 회원권 시작일입니다.

`endDate`: 회원권 만료일입니다.

`paymentAmount`: 결제금액입니다.

`paymentStatus`: 결제 상태입니다. `완료` 또는 `미납`을 사용합니다.

`attendanceDates`: 출석한 날짜 목록입니다.

### 확장 데이터 구조

프로젝트가 커지면 아래처럼 데이터를 나누면 좋습니다.

```js
Member
- id
- name
- phone

Membership
- id
- memberId
- type
- startDate
- endDate

Attendance
- id
- memberId
- date

Payment
- id
- memberId
- amount
- status
- paidAt
```

초기 개인 프로젝트에서는 단순한 구조로 시작하고, 기능이 늘어나면 분리하는 방식이 좋습니다.

## 6. 기능별 CRUD 정리

CRUD는 데이터를 다루는 기본 동작입니다.

- Create: 새로 만들기
- Read: 조회하기
- Update: 수정하기
- Delete: 삭제하기

### 회원 관리 CRUD

Create:

- 새 회원을 등록합니다.
- 이름, 연락처, 회원권, 결제 정보를 입력합니다.

Read:

- 등록된 회원 목록을 조회합니다.
- 이름이나 연락처로 회원을 검색합니다.

Update:

- 회원 정보를 수정합니다.
- 회원권 종류, 만료일, 결제금액 등을 변경합니다.

Delete:

- 더 이상 관리하지 않는 회원을 삭제합니다.

### 회원권 관리 CRUD

Create:

- 회원에게 새 회원권을 등록합니다.

Read:

- 회원권 종류, 시작일, 만료일을 확인합니다.
- 만료일까지 남은 날짜를 확인합니다.

Update:

- 회원권을 연장하거나 만료일을 수정합니다.

Delete:

- 잘못 등록된 회원권 정보를 삭제합니다.

### 출석 관리 CRUD

Create:

- 오늘 출석 기록을 추가합니다.

Read:

- 최근 출석일과 총 출석 횟수를 확인합니다.

Update:

- 잘못 체크한 출석 날짜를 수정합니다.

Delete:

- 실수로 체크한 출석 기록을 취소합니다.

### 결제 관리 CRUD

Create:

- 결제 정보를 등록합니다.

Read:

- 결제금액과 결제상태를 확인합니다.

Update:

- 미납 상태를 완료 상태로 변경합니다.
- 결제금액을 수정합니다.

Delete:

- 잘못 등록된 결제 정보를 삭제합니다.

### 만료일 관리 CRUD

Create:

- 회원권 등록 시 시작일과 만료일을 저장합니다.

Read:

- 만료일까지 남은 날짜를 확인합니다.

## Git 버전 관리 가이드

Git은 프로젝트의 변경 기록을 저장하는 도구입니다. 기능을 하나 만들 때마다 저장해두면, 언제 어떤 작업을 했는지 확인할 수 있고 문제가 생겼을 때 이전 상태로 돌아가기 쉽습니다.

### 1. 프로젝트 폴더 생성

```bash
mkdir gym-member-system
cd gym-member-system
```

현재처럼 이미 폴더가 있다면 해당 폴더로 이동하면 됩니다.

```bash
cd C:\project
```

### 2. Git 시작

```bash
git init
```

이 명령어를 실행하면 현재 폴더가 Git으로 관리되기 시작합니다.

### 3. 파일 생성

예시 파일:

```bash
README.md
index.html
styles.css
app.js
```

역할:

- `README.md`: 프로젝트 설명서
- `index.html`: 화면 구조
- `styles.css`: 디자인
- `app.js`: 기능 동작

### 4. 상태 확인

```bash
git status
```

현재 변경된 파일과 아직 Git에 저장하지 않은 파일을 확인합니다.

### 5. 파일 추가

```bash
git add README.md
git add index.html styles.css app.js
```

모든 변경 파일을 한 번에 추가하려면 아래 명령어를 사용할 수 있습니다.

```bash
git add .
```

### 6. 커밋 생성

```bash
git commit -m "docs: add project blueprint"
```

커밋은 현재 작업 내용을 하나의 기록으로 저장하는 동작입니다.

### 7. 커밋 기록 확인

```bash
git log --oneline
```

최근 커밋 목록을 짧게 확인할 수 있습니다.

## 커밋 메시지 예시

기능 하나를 완성할 때마다 아래처럼 메시지를 작성하면 좋습니다.

```bash
git commit -m "docs: add project blueprint"
git commit -m "feat: add member registration form"
git commit -m "feat: render member list"
git commit -m "feat: add member search"
git commit -m "feat: add attendance check"
git commit -m "feat: add payment status toggle"
git commit -m "feat: calculate membership expiration"
git commit -m "style: update soft light dashboard theme"
git commit -m "fix: prevent invalid expiration date"
```

커밋 메시지 앞 단어의 의미:

- `docs`: 문서 작업
- `feat`: 새로운 기능
- `fix`: 오류 수정
- `style`: 디자인 또는 코드 스타일 변경
- `refactor`: 기능 변화 없이 코드 구조 개선

초보자라면 처음에는 너무 어렵게 생각하지 않아도 됩니다. 중요한 것은 “무엇을 바꿨는지 짧고 명확하게 적는 것”입니다.
